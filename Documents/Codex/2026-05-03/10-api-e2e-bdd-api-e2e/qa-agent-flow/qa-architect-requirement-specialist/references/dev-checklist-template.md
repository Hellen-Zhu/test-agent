# 开发用需求测试点 Checklist 模板 (Dev-Facing Requirement Test Checklist)

本模板用于在需求分析完成后，**同步输出一份可直接给开发看的测试点清单**，便于开发自测、联调验收与需求对照。输出时可采用本结构，保持「可勾选、按模块/来源追溯」的形式。

---

## 文档头（必填）

| 项目 | 内容 |
|------|------|
| 需求/模块名称 | （与需求分析文档一致） |
| 分析日期 | YYYY-MM-DD |
| 需求来源 | 图1～图N / PRD 章节 / 链接（若有） |

---

## 一、功能与界面（按图/模块分节）

**说明**：若需求来自多图，按「图1」「图2」…分小节；每条测试点标注来源（如图号），便于对照。

## 使用说明

- **开发自测**列：供开发或联调方勾选，表示已自测通过。
- **来源**列：填写图号（图1、图2）或 PRD 章节，便于追溯需求。
- 本 Checklist 由需求分析结论提炼而来，与《需求分析完整文档》中的【深度测试点】及专家 Checklist 结论一致；若需求有变更，需同步更新本文档与需求分析文档。
